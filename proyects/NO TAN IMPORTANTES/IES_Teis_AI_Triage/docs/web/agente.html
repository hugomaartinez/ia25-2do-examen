Cómo el Agente interactúa con el RAG

01

Análisis

El agente recibe el email del cliente y extrae la intención y palabras clave para construir la consulta.

02

Recuperación

Busca en ChromaDB los fragmentos de FAQ más similares semánticamente mediante embeddings vectoriales.

03

Síntesis

El LLM genera una respuesta natural combinando el contexto recuperado con el prompt de sistema.

Prompt de Sistema

El Prompt de Sistema es la directiva de comportamiento del agente. Define su identidad, restricciones y el protocolo de escalado. Su redacción es crítica.

System Prompt
"Eres un agente de soporte de TechPyme.
                Responde preguntas de clientes usando solo el contexto
                de las FAQs que se te proporciona.

                Si la información solicitada NO está en el contexto,
                o si el tono del cliente es agresivo o contiene insultos,
                responde exactamente con la etiqueta:
                    [ESCALADO_HUMANO]

                No inventes información. No especules.
                Contexto: {context}
                Pregunta del cliente: {question}"

Gestión de Alucinaciones

Las alucinaciones ocurren cuando el LLM inventa información no respaldada por el contexto. Dos mecanismos clave las mitigan:

Contexto Cerrado

Técnica: Grounding

La instrucción "usando solo el contexto" le prohíbe al LLM usar su conocimiento preentrenado. Si la FAQ no contiene la respuesta, el agente debe escalar, no inventar.

Temperatura = 0.0

Parámetro Crítico

Con temperature=0.0 el modelo siempre escoge el token más probable. Respuestas deterministas y predecibles. Nunca uses temperatura alta en agentes de producción.

0.0 · Preciso1.0 · Creativo2.0 · Caótico

Protocolo de Escalado Humano

Criterios de decisión para derivar el caso

Insultos o Agresividad

El agente detecta lenguaje hostil y transfiere automáticamente a un operador para preservar la relación con el cliente.

Información No Encontrada

La pregunta no tiene correspondencia en el RAG. El agente no inventa: escala antes que dar una respuesta incorrecta.

Casos Complejos o Legales

Reclamaciones, devoluciones complejas o temas legales superan la capacidad del agente y requieren criterio humano.

Cliente VIP o Recurrente

Clientes identificados como prioritarios pueden ser derivados siempre a un agente humano por política comercial.

El agente activa el escalado devolviendo la etiqueta exacta [ESCALADO_HUMANO] que el sistema Python detecta en tiempo real.
