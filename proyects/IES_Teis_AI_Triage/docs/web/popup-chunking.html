Chunking  (Fragmetación)

Cómo dividir documentos extensos para que tu IA recupere la información exacta sin perder el contexto.

¿Qué es el Chunking?

En un sistema RAG, no podemos inyectar un libro entero en el prompt de una vez. El Chunking es el proceso de dividir un documento en fragmentos más pequeños y manejables.

Estos fragmentos se convierten en vectores (embeddings) y se almacenan en una base de datos. Cuando el usuario pregunta algo, el sistema busca los chunks más relevantes semánticamente.

Estrategias de Chunking

1. Chunking de Tamaño Fijo (Fixed-size Chunking)

Es el método más básico. Se define un número exacto de caracteres o tokens por fragmento.

  • Cómo funciona: Si decides fragmentos de 500 caracteres, el sistema corta cada 500.
  • Pros: Muy rápido y computacionalmente barato.
  • Contras: Puede cortar una frase o una palabra por la mitad, perdiendo el sentido completo.
  • Solución: Se suele usar un overlap (solapamiento), donde el final del fragmento 1 se repite al inicio del fragmento 2 para mantener el contexto.

2. Fragmentación Recursiva (Recursive Character Splitting)

Es el estándar actual para documentos de texto plano (usado por herramientas como LangChain).

  • Cómo funciona: Intenta dividir el texto por jerarquía de separadores: primero busca párrafos (\n\n), luego saltos de línea (\n), luego espacios entre palabras ( ), y finalmente caracteres individuales si es necesario.
  • Objetivo: Mantener los párrafos y oraciones unidos tanto como sea posible dentro del límite de tamaño definido.

3. Chunking Basado en Documentos (Structure-aware)

Este método respeta la estructura lógica del archivo original.

  • Markdown: Divide por encabezados (#, ##, ###).
  • HTML: Divide por etiquetas de sección o artículos (<section>, <article>).
  • Código: Divide por funciones o clases.
  • Tablas: Requiere un tratamiento especial para no perder la relación entre filas y columnas.

4. Chunking Semántico (Semantic Chunking)

Es el método más avanzado y reciente. Divide el texto basándose en el significado (semántica) en lugar de la estructura.

  • Cómo funciona: Utiliza modelos de embeddings (vectores) para detectar cambios de tema en el texto. Busca dónde se "distancia" semánticamente el contenido y corta ahí.
  • Ventaja: Evita cortar frases a la mitad porque no corta en cualquier lugar, sino donde cambia el contexto.
  • Desventaja: Es computacionalmente más costoso que los métodos anteriores.
  • Uso: Ideal para documentos técnicos complejos o textos donde la estructura no sigue reglas claras (ej. transcripciones de reuniones).

El Secreto: El Overlap (Solapamiento)

Si cortamos un texto por la mitad, podemos perder la relación entre el sujeto y el predicado. Por eso, repetimos un poco de texto entre fragmentos.

Chunk 01
"El cliente tiene un plazo de 14 días para devolver el productoen su embalaje original."
Chunk 02
"en su embalaje original. Una vez recibido, el reembolso tardará 5 días."

Reglas de Oro del Chunking

01

¿Tamaño?

  • FAQs: 128 a 256 tokens (coincide naturalmente con pares de pregunta-respuesta)
  • Documentos técnicos: 300 a 500 tokens (preserva los procedimientos paso a paso)
  • Artículos de investigación: 512 a 1024 tokens (mantiene argumentos complejos)
  • Documentos legales: 600 a 1000 tokens (conserva la integridad de las cláusulas)
  • Documentación de código: 200 a 400 tokens (se alinea con el contexto a nivel de función)
02

¿Overlap?

Un 10% a 20% de solapamiento es el estándar. Garantiza que la transición entre fragmentos sea coherente.

03

¿Modelo?

El tamaño del chunk debe ser menor que la ventana de contexto de tu modelo de Embeddings (ej: Llama 3 tiene 8k).
